Você está trabalhando em um app web estático chamado Flui360, feito em HTML, CSS e JavaScript puro, com as páginas index.html, dashboard.html, habitos.html, relatorios.html, um CSS principal e um main.js.

Preciso que você refine o projeto pensando na avaliação de uma disciplina de Interação Humano-Computador (IHC). Não mude a ideia geral do app, apenas melhore a usabilidade, acessibilidade e clareza do fluxo.

Aplique as seguintes melhorias:

Navegação e fluxo

Destacar na sidebar qual é a página atual, adicionando uma classe (por exemplo .nav-link-ativo) e, se possível, aria-current="page" no item correspondente. Estilize essa classe no CSS com cor de fundo ou borda lateral.

No dashboard.html, adicionar um pequeno texto de boas-vindas explicando que o usuário começa com hábitos de exemplo e que pode criar/editar hábitos usando o botão +.

Em adicionar um hábito novo, adicionar um texto curto explicando a diferença entre hábito binário (Sim/Não) e hábito mensurável

Feedback e estado do sistema

Ao salvar um hábito (criação ou edição), exibir um feedback visual curto para o usuário, por exemplo um toast ou uma mensagem temporária na tela (“Hábito salvo com sucesso”), sem recarregar a página.

Ao excluir um hábito, após a confirmação, mostrar uma mensagem rápida (“Hábito excluído”).

Ao registrar o valor de um hábito mensurável (via modal), mostrar uma mensagem rápida (“Progresso registrado!”).

Garantir que, no formulário de hábito, campos obrigatórios sejam validados (nome, tipo, alvo quando mensurável, X/Y quando necessário) e que mensagens de erro amigáveis sejam exibidas próximos aos campos.

Acessibilidade

Substituir elementos clicáveis que hoje são <div onclick="..."> (por exemplo, controles de completar hábito na lista e na seção “Hábitos de Hoje”) por elementos <button type="button">, mantendo o mesmo estilo visual via CSS. Isso deve valer tanto para hábitos binários quanto para os botões que abrem o modal de valor dos mensuráveis.

Ao abrir qualquer modal (criar/editar hábito, valor mensurável, confirmação de exclusão, etc.), colocar o foco no primeiro elemento relevante (por exemplo, o campo de nome ou botão principal).

Implementar o fechamento dos modais pela tecla Esc (Escape), além dos botões já existentes.

Conferir rapidamente o contraste de textos principais com o fundo. Se algum texto estiver muito claro, ajustar a cor para um cinza mais escuro.
